# Legends Live On

This project is a premium podcast and spiritual community website **Legends Live On**.

## Overview

- **Brand Name:** "Legends Live On"
- **Service Professionalization:**
  - Podcast Episodes -> **Profound Narrative Episodes**
  - Mystery Merch -> **Legacy Collective Merchandise**
  - Spiritual Reflections -> **Eternal Truth Insights**
  - Sunday Service -> **Sacred Fellowship Gatherings**
  - Blog Posts -> **Faith & Heritage Chronicles**
  - Community -> **Global Kingdom Community**

## Project Structure

- `index.html`: Homepage with a cinematic experience, highlighting latest episodes and spiritual reflections.
- `aboutus.html`: The mission and the heart behind the Legends Live On community.
- `episodes.html` & `episode-detail.html`: A library of immersive podcast episodes exploring truth, grace, and human stories.
- `blog.html` & `blog-details.html`: Articles and chronicles focused on faith, heritage, and spiritual growth.
- `products.html` & `product-detail.html`: A curated shop for mystery merch and legacy collective items.
- `cart.html`, `checkout.html` & `account.html`: Full e-commerce flow for supporting the mission through merchandise.
- `contactus.html`: Portal for inquiries, prayer requests, and community connection.
- `layout/`: Shared components including `header.html` and `footer.html`.
- `css/`: Custom styling architecture including `custom.css`, `responsive.css`, and `css-new.css`.
- `js/`: High-end interactivity and animations, including GSAP integrations for a premium feel.

## Technologies Used

- **HTML5/CSS3:** Semantic structure with custom, high-fidelity layouts.
- **Bootstrap:** Robust, responsive grid system.
- **GSAP (GreenSock):** Advanced animations for a sophisticated, cinematic user experience.
- **jQuery:** Seamless DOM manipulation and slider integrations.
- **AOS (Animate On Scroll):** Dynamic element transitions for engaging storytelling.

## Installation & Setup

1. Clone the repository.
2. Open `index.html` in any modern web browser.
3. No server-side dependencies are required for front-end preview.
